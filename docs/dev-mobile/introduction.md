# Introduction au développement mobile

Le développement mobile consiste à créer des applications destinées aux **smartphones** et **tablettes**.  
Ces applications doivent tenir compte de contraintes spécifiques : **petits écrans**, **interaction tactile**, **ressources limitées** et **écosystèmes fermés** (Android, iOS).

## Un peu d’histoire

- Avant 2007 : applications Java ME, Symbian, Blackberry, PalmOS. Expérience limitée, écosystèmes fragmentés.
- 2007 : iPhone → révolution de l’expérience utilisateur et lancement de l’App Store en 2008.
- 2010 : Android explose avec un écosystème plus ouvert.
- Aujourd’hui : Android + iOS = plus de 95% du marché mobile.

## Cycle de vie d’un projet mobile

1. **Analyse & scénario** : besoins utilisateurs, données nécessaires, contraintes techniques.
2. **Conception** : maquettes, navigation, choix de persistance.
3. **Développement** : choix techno, mise en place du projet, implémentation incrémentale.
4. **Tests** : fonctionnels (besoins couverts ?) et non-fonctionnels (ergonomie, performance, sécurité, compatibilité devices).
5. **Publication** : préparation de la build, vérification des contraintes des stores, suivi des mises à jour.

## Écosystème actuel

- **Android** : système ouvert, grande diversité d’appareils, langage natif Java/Kotlin.
- **iOS** : système fermé, matériel contrôlé, langage natif Swift/ObjC.
- **Frameworks cross-platform** : Ionic, React Native, Flutter → un seul code pour Android + iOS.
- **PWA** : applications web enrichies pouvant être installées et fonctionner offline.

## Contraintes spécifiques au mobile

- **Écran** : taille réduite, orientation portrait/paysage, densité variable (DPI).
- **Interaction** : tactile, gestes (swipe, pinch), clavier virtuel.
- **Ressources** : batterie limitée, réseau fluctuant, stockage restreint.
- **Sécurité** : gestion des permissions (caméra, GPS), données sensibles, sandboxing.
